# tg-mod-bot

Telegram-бот для модерации группы/чата: антиспам, антиреклама, флуд-контроль,
проверка новых участников (капча) и админ-команды. Python 3.12 + aiogram 3, Docker.

## Возможности

- **Антиспам/антиреклама** — удаляет сообщения со ссылками (кроме доменов из
  вайтлиста) и сообщения с запрещёнными словами/фразами; проверяет автора по
  бан-листу [CAS](https://cas.chat) и banит сразу, если он там числится.
  Накопление предупреждений → автобан по лимиту.
- **Флуд-контроль** — если пользователь шлёт больше N сообщений за M секунд,
  бот мьютит его на заданное время (Telegram сам снимает ограничение по таймеру).
- **Проверка новых участников** — при входе в чат участник получает ограничение
  на отправку сообщений и кнопку «Я не бот». Не нажал за отведённое время — кик.
- **Анти-рейд** — при массовом входе (`RAID_JOIN_LIMIT` человек за
  `RAID_WINDOW_SECONDS` секунд) включается локдаун: новых участников сразу
  ограничивают, админам приходит алерт.
- **Админ-команды** (только для админов чата или ID из `ADMIN_IDS`):
  `/ban /unban /kick /mute /unmute /warn /unwarn /warns /id /modhelp`
  (все, кроме `/unban`, отправляются в ответ на сообщение пользователя).
- **Пресеты бана** — `/ban 1` и `/ban 2` банят пользователя и публикуют
  заготовленный текст (редактируется в панели, без правки кода).
- **`/check`** — в ответ на сообщение пользователя публикует текст-предупреждение
  с дедлайном **+2 часа по Пермскому времени (UTC+5)**, а админу в ЛС приходит
  уведомление с кнопками «Заблокировать» / «Ограничить доступ».
- **Уведомления в ЛС** — все события бот дублирует в личку каждому админу из
  `ADMIN_IDS` (нужно один раз нажать Start в диалоге с ботом).
- **Админ-панель `/panel`** (в личке бота) — редактирование текстов пресетов и
  `/check`, включение/выключение антиспама и антирейда на лету. Настройки хранятся
  в БД, рестарт не нужен.
- Все действия при заданном `LOG_CHAT_ID` дополнительно дублируются в лог-чат.

### Модель доступа (важно!)

Власть над ботом и иммунитет от модерации есть **только** у Telegram ID из
`ADMIN_IDS`. Статус «администратор чата» **не даёт никаких прав** — если у тебя в
чате есть админы-рекламщики, они модерируются наравне со всеми. Вписывай в
`ADMIN_IDS` только тех, кому реально доверяешь.

### Лимит одинаковых сообщений

Один и тот же текст от пользователя разрешён `DUPLICATE_LIMIT` раз (по умолчанию 3)
за `DUPLICATE_WINDOW_HOURS` часов — дальше копии удаляются автоматически.

⚠️ Telegram **запрещает боту удалять сообщения и банить администраторов чата**.
Если рекламщик — админ чата, бот не сможет удалить его дубли и вместо этого
пришлёт тебе в ЛС предупреждение с рекомендацией снять с него права. Чтобы лимит
работал в полную силу — **забери у рекламщиков админку в чате**.

### Защита от случайных банов

- `ban`/`mute`/`kick` **никогда** не применяются к `ADMIN_IDS` (двойная проверка).
- Все действия — best-effort: ошибка API (нет прав, цель вышла) не роняет бота.
- Анти-рейд только временно ограничивает (мьют), а не банит.
- Короткие сообщения не считаются повторами (порог `DUPLICATE_MIN_LENGTH`).

## Быстрый старт (VPS, Docker)

```bash
git clone <это репо> tg-mod-bot   # или скопируй папку на сервер
cd tg-mod-bot
./deploy.sh <BOT_TOKEN> <ADMIN_IDS>   # ADMIN_IDS через запятую, необязателен
```

Скрипт сам поставит Docker (если его нет), создаст `.env` из `.env.example`,
подставит токен и поднимет бота: `docker compose up -d --build`.

Если запустить `./deploy.sh` без аргументов и `.env` ещё нет — он будет создан
из примера, и скрипт попросит вписать `BOT_TOKEN` вручную перед повторным запуском.

## Настройка

Все параметры — в `.env` (см. комментарии в `.env.example`):
токен бота, список ID модераторов, чат для логов, пороги флуда, лимит
предупреждений, таймаут капчи, список запрещённых слов, вайтлист доменов.

После правки `.env`:

```bash
docker compose up -d --build
```

## Права бота в чате

Бот должен быть администратором чата с правами: удаление сообщений,
бан/ограничение участников, закрепление (если используется).

## Логи

```bash
docker compose logs -f
```

## Локальный запуск без Docker

```bash
python -m venv .venv && . .venv/Scripts/activate   # Windows
pip install -r requirements.txt
cp .env.example .env   # и заполнить BOT_TOKEN
python -m bot.main
```
